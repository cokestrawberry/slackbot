# PRD — 스프린트 통계 (`@지라 통계`)

> **Branch:** `feature/sprint-statistics`
> **우선순위:** 중간
> **난이도:** 중간

## 1. 배경

현재 `@지라 scrum`은 담당자별 이슈 목록을 보여주지만, **SP 기반 통계 요약**이 없다.
"오늘 얼마나 해결했는지", "전체 대비 진척률", "가장 큰 이슈가 뭔지" 등을 한눈에 파악하고 싶다.

## 2. 사용자 시나리오

### 2-1. 통계 요약
```
사용자: @지라 통계

📊 스프린트 통계 요약

🔥 진척률
  전체: 45 SP | 완료: 20 SP | 남음: 25 SP
  ████████░░░░░░░░░░░░ 44%

📋 상태별 현황
  ✅ 완료: 5건 (20 SP)
  🔨 진행 중: 3건 (12 SP)
  📋 해야 할 일: 4건 (13 SP)

🏆 오늘 해결된 이슈
  • SLAC-15 결제 금액 0원 표시 버그 (SP 5, 담당: 김영현)
  • SLAC-22 세션 만료 무한 로딩 (SP 3, 담당: 김영현)
  → 오늘 8 SP 완료!

🔨 현재 진행 중
  • SLAC-7 로그인 500 에러 (SP 2, 담당: 김영현)
  • SLAC-30 다크모드 지원 (SP 5, 담당: 최아록)
  • SLAC-31 알림 설정 페이지 (SP 5, 담당: 최아록)

📌 가장 큰 이슈 (미완료)
  • SLAC-28 결제 시스템 리팩토링 (SP 8, 해야 할 일, 담당: 김영현)
```

### 2-2. 이슈가 없는 경우
```
사용자: @지라 통계
봇: DB에 이슈가 없습니다. `@지라 sync`로 동기화해주세요.
```

### 2-3. 오늘 해결된 이슈가 없는 경우
```
🏆 오늘 해결된 이슈
  (없음)
```

## 3. 기술 설계

### 3-1. 라우팅 (SlackEventController)

**키워드 매칭:**
```
"통계", "stats", "statistics" → handleStatistics(event)
```

**Haiku fallback:**
- 기존 `statistics` intent → `handleStatistics()` 호출로 변경 ("준비 중" 제거)

### 3-2. 진척률 프로그레스 바

텍스트 기반 프로그레스 바 (20칸):
```java
String progressBar(double ratio) {
    int filled = (int) (ratio * 20);
    return "█".repeat(filled) + "░".repeat(20 - filled);
}
// 44% → ████████░░░░░░░░░░░░ 44%
```

### 3-3. "오늘 해결된 이슈" 기준

- `completedAt`이 오늘 날짜 (KST 기준)에 해당하는 이슈
- `completedAt`이 null이면 `jiraUpdated`가 오늘이고 `statusCategory == '완료'`

```java
Instant todayStart = LocalDate.now(ZoneId.of("Asia/Seoul"))
    .atStartOfDay(ZoneId.of("Asia/Seoul")).toInstant();
```

### 3-4. DB 쿼리

기존 `issueRepository.findAll()` 사용 후 메모리에서 필터링.
이슈 수가 수백 건 이내이므로 별도 쿼리 불필요.

### 3-5. 영향 범위

| 파일 | 변경 내용 |
|------|-----------|
| `SlackEventController.java` | `routeCommand()`에 `통계` 키워드 추가, `handleStatistics()` 메서드, `handleWithIntent()`의 `statistics` case 수정 |
| `ScrumReportService.java` / `Impl` | `generateStatisticsReport()` 메서드 추가 |
| `HELP_TEXT` | 통계 명령 추가 |

### 3-6. 새 파일: 없음

`ScrumReportServiceImpl`에 메서드 추가. 기존 리포트 로직과 유사한 패턴.

## 4. 출력 구성 요소

| 섹션 | 내용 | 데이터 소스 |
|------|------|-------------|
| 🔥 진척률 | 전체 SP, 완료 SP, 남은 SP, 프로그레스 바 | 전체 이슈 SP 합산 |
| 📋 상태별 현황 | 상태별 이슈 수 + SP 합계 | statusCategory 그룹핑 |
| 🏆 오늘 해결 | 오늘 완료된 이슈 목록 + 오늘 SP 합계 | completedAt >= 오늘 시작 |
| 🔨 진행 중 | 현재 진행 중인 이슈 목록 | statusCategory == '진행 중' |
| 📌 가장 큰 이슈 | 미완료 중 SP가 가장 큰 이슈 1건 | statusCategory != '완료' 중 SP max |

## 5. 번업 차트

Slack은 이미지/차트를 직접 렌더링할 수 없으므로 **텍스트 기반 간이 차트**로 대체:

```
📈 번업 (최근 7일)
  05/06 ██████████████████░░ 35/45 SP
  05/07 ██████████████████░░ 35/45 SP
  05/08 ████████████████████ 37/45 SP
  05/09 ████████████████████ 38/45 SP
  05/10 ████████████████████ 40/45 SP
  05/11 ████████████████████ 40/45 SP
  05/12 ████████████████████ 42/45 SP
```

**구현 방식:**
- DB에 일별 완료 SP 이력이 없으므로, `completedAt` 기반으로 역산
- 각 날짜 시점까지 완료된 SP 누적 합계 계산
- 전체 SP는 현재 값 고정 (과거 이슈 추가/삭제 반영 어려움 → 근사치)

**한계:** 스프린트 중간에 이슈가 추가/제거된 경우 전체 SP가 변동되지만 반영 불가. "참고용 근사치"로 제공.

## 6. 엣지 케이스

| 케이스 | 처리 |
|--------|------|
| DB 비어있음 | "sync 먼저 실행" 안내 |
| 모든 이슈 완료 | 진척률 100%, 진행 중/해야 할 일 섹션 생략 |
| SP가 0인 이슈만 | 진척률은 이슈 수 기준으로 fallback |
| completedAt이 null인 완료 이슈 | jiraUpdated 사용 |
| 오늘 해결 이슈 없음 | "(없음)" 표시 |

## 7. 테스트 계획

### 단위 테스트
- 진척률 계산 (0%, 50%, 100%)
- 프로그레스 바 렌더링
- 상태별 그룹핑 + SP 합산
- 오늘 완료 이슈 필터링 (KST 기준 날짜 경계)
- 번업 차트 데이터 역산
- 가장 큰 이슈 선택

### 통합 테스트
- `@지라 통계` → 스레드 응답 확인
